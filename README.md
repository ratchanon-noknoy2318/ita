## About

This project documents my experience working with a **legacy PHP 5.0+ healthcare system** during an internship.

The system had **no proper documentation**, and I was responsible for diagnosing issues affecting real hospital workflows.

I encountered an **HTTP 404 error on the production interface**, then traced the request flow to identify the broken routing path in an OPD module.

The issue was analyzed and resolved within approximately **8 hours**, restoring normal workflow operations.

---

## What I Did

- Traced request flow in a legacy PHP architecture
- Identified broken or misconfigured endpoints
- Fixed routing issues without breaking existing hospital operations
- Verified changes against real workflow usage

---

## What I Learned

- Legacy systems require careful, incremental debugging
- Logs and request flow tracing are critical without documentation
- Small routing issues can impact real clinical operations
- Production systems often prioritize stability over clean architecture

---

## Impact

- Restored broken endpoints in OPD workflow module, ensuring continuity of hospital operations
- Improved system reliability for daily clinical and administrative workflows
- Reduced manual workarounds for staff during patient processing
- Prevented unnecessary redevelopment cost (~9,000 THB) by resolving issues within the existing system instead of external code replacement
- Stabilized the system under time-sensitive operational conditions
